## 360V6 路由器 OpenWrt 部署方案

### 背景

原厂固件的无线 Mesh 组网体验一般，网上现成的 OpenWrt 固件也不太符合需求。借助 AI 定制了一个仅包含 HomeProxy 的最简固件，体验良好，后续希望进一步压榨这台路由器的性能。

### 方案设计

经过多次测试，采用**分级部署策略**，按 U 盘就绪状态动态加载组件：

| 组件 | 存储位置 | 触发条件 | 特性 |
|------|----------|----------|------|
| **HomeProxy** | Flash (/overlay) | 始终安装 | 固件内置，基础科学上网 |
| **Docker** | U 盘 ($_USB/docker-*) | 检测到已挂载 U 盘 | 按需加载，支持多版本降级 |
| **AdGuardHome** | U 盘 ($_USB/adguardhome-*) | U 盘已挂载 + 下载成功 | 多源轮询下载，支持断点续传 |
| **Swap** | U 盘 ($_USB/swapfile) | 文件系统支持 + 内存 < 256MB | 自动创建 512MB，写入验证，坏块自动规避 |

### 核心特性

| 特性 | 说明 |
|------|------|
| ✅ 状态随盘迁移 | 换盘/重插/设备号漂移不会丢失状态 |
| ✅ 减少 Flash 写入 | 状态文件存放于 U 盘，大幅降低 Flash 磨损 |
| ✅ 一键迁移 | 首次运行时自动将旧状态从 Flash 迁至 U 盘 |
| ✅ 精简存储 | Flash 仅保留三项与 USB 无关的状态 |

### 管理命令

| 命令 | 功能 |
|------|------|
| `360v6-status` | 状态报告 |
| `360v6-health [--fix]` | 健康检查 / 自动修复 |
| `360v6-format-usb [--yes]` | 格式化 U 盘为 ext4 |
| `360v6-tune --mode=...` | 性能调优（performance / balanced / powersave）|
| `360v6-upgrade --docker\|--agh` | 升级 Docker 或 AdGuardHome 组件 |
| `agh-finalize` | AGH 初始化（完成向导后执行） |
| `360v6-uninstall [--yes]` | 卸载全部组件 |
